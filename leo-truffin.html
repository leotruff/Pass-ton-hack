Problematique 1 - Organiser un depot propre pour les preuves

Enonce / Descriptif

Contexte : centraliser les preuves des challenges dans un depot commun. Objectif : obtenir une structure claire, partageable, et facile a relire en soutenance.

Resolution

  1. Creation des dossiers standards `assets/eleves/` par membre.
  2. Definition d'une convention de nommage des captures (`p1_01.png`, etc.).
  3. Mise en place d'une page HTML par eleve avec sections obligatoires.
  4. Verification finale des liens et coherence des preuves.

Outil(s) / Ressource(s) utilisees

  • Git et GitHub
  • VS Code
  • HTML5 semantique

Difficulte(s) rencontree(s) + correction

Difficulte : certains noms de fichiers n'etaient pas uniformes. Correction : adoption d'un schema unique par problematique et verification collective.

Ce que j'ai appris

  • Structurer un projet web collaboratif avec des conventions strictes.
  • Rendre des preuves lisibles et exploitables pour un evaluateur externe.
  • Identifier vite les erreurs de liens avant livraison.

Grand Oral : ce que je peux reutiliser

Je peux reutiliser cette methode pour expliquer la gestion de projet, la qualite documentaire, et la coordination technique d'une equipe.

Commandes utilisees

# Initialisation et controle du depot
git status
mkdir -p assets/eleves/leo-truffin
rg --files

Problematique 2 - Consolider la qualite finale avant remise

Enonce / Descriptif

Contexte : avant la remise, il faut eviter les erreurs de format. Objectif : verifier conformite HTML/CSS et coherences de contenu sur toutes les pages.

Resolution

  1. Relecture complete de chaque page eleve.
  2. Controle du nombre de titres et de la semantique `header/main/section/footer`.
  3. Verification des chemins d'images `assets/eleves/prenom-nom/`.
  4. Correction des derniers details de lisibilite CSS.

Outil(s) / Ressource(s) utilisees

  • Ripgrep pour controles rapides
  • Navigateur pour test responsive
  • Checklist projet

Difficulte(s) rencontree(s) + correction

Difficulte : oublis de sections sur certaines pages. Correction : application d'un gabarit unique pour garantir la conformite.

Ce que j'ai appris

  • Un audit final evite la majorite des erreurs de rendu.
  • La semantique HTML facilite la lecture du dossier.
  • Une checklist claire accelere la validation equipe.

Grand Oral : ce que je peux reutiliser
